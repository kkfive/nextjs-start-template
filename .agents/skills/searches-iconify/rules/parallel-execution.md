# 并行执行规则（硬约束）

所有搜索请求**必须在同一响应中并发发起**。

## 正确

```
单次响应中同时发起：
  curl ... &
  curl ... &
  curl ... &
  wait
```

或在 Agent 体系中：**单条 message 里包含多个 Bash tool 调用**。

## 错误

```
响应 1：curl settings
响应 2：curl gear
响应 3：curl cog
```

每次响应间用户都要等待 + Agent 上下文累积，体验差且慢。

## 为什么必须并行

- Iconify API 单请求 < 300ms；串行 5 个 = 1.5s+，并行 = 300ms
- 用户感知到一次性结果，而非"一点一点"
- 减少 Agent 与用户的轮次开销

## 等待全部完成再聚合

```bash
# 不要拿到第一个就开始输出 —— 会让聚合排序失真
# 等所有 wait 完成 → 再聚合 → 再输出
```

## 检查

- [ ] 所有关键词的 curl 在同一响应触发
- [ ] 不在中间穿插其他工具调用
- [ ] 等全部完成后才进入聚合
- [ ] 输出按聚合后的顺序，不按关键词到达顺序
